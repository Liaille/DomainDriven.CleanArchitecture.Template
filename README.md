# DomainDriven.CleanArchitecture.Template
# .NET 基础设施全量手册
本文档汇总.NET项目基础设施核心能力，涵盖基础架构、安全认证、日志监控、数据访问等28个模块，作为项目架构设计、开发规范的参考手册。
## 一、基础架构与应用托管
### 应用启动与宿主配置
- 统一 Host 构建（WebApplication / Host.CreateDefaultBuilder）
- 多环境区分（Development/Test/Staging/Production），环境变量优先级配置
- 模块化服务注册（Composition Root 模式），统一入口组装所有依赖
- 应用生命周期管理（启动、运行、关闭、异常退出处理）
### 强类型配置绑定
- 基于 IOptions<T> / IOptionsSnapshot / IOptionsMonitor 实现配置注入
- 配置启动时校验（数据类型、必填项、取值范围），校验失败阻止启动
- 支持配置中心接入（Nacos/Apollo/Consul），配置热更新、灰度发布
- 敏感配置加密存储（如数据库连接串、密钥），解密注入
### 模块化扩展机制
- 每层独立扩展方法：AddApplication ()、AddInfrastructure ()、AddPresentation ()
- 插件式架构支持，可动态加载 / 卸载模块
- 模块依赖管理，避免循环依赖
### 全局请求上下文
- 当前用户上下文（ICurrentUser）：获取用户 ID、角色、权限等信息
- 当前租户上下文（ICurrentTenant）：租户 ID、租户配置自动获取
- 请求生命周期上下文：存储请求 ID、链路 ID、请求时间等，贯穿整个请求链路
- 上下文异步安全存取，支持异步线程间传递
## 二、安全与认证授权
### JWT 认证
- JWT 令牌颁发、验证、刷新、吊销机制
- 令牌携带用户信息、角色、权限，微服务间身份无缝传递
- 令牌过期策略、刷新令牌机制，避免频繁登录
- 支持令牌加密签名，防止篡改
### OAuth2 / OIDC 集成
- 适配 IdentityServer、Azure AD、Auth0 等第三方认证服务
- 授权码模式、客户端凭证模式支持，满足不同场景（前端登录、服务间调用）
- 统一认证回调处理，用户信息映射
### RBAC 权限体系
- 角色（Role）、权限（Permission）、资源（Resource）三级权限模型
- 权限与角色关联、角色与用户关联，支持多角色分配
- 权限粒度控制（接口级、按钮级、数据级）
### Policy 授权策略
- 基于 Policy 的动态权限校验，支持自定义授权逻辑
- 权限策略与接口绑定，无需在业务代码中嵌入权限判断
- 支持匿名访问、角色授权、权限授权、自定义条件授权
### API 签名验证
- 基于 AppKey + AppSecret + 时间戳 + 随机串 + 签名的接口验证机制
- 签名过期控制，防止签名被复用
- 支持微服务内部调用、第三方调用的签名校验
### 防重放攻击
- Nonce（随机串）+ Timestamp（时间戳）双重校验，防止请求重放
- Nonce 缓存机制，避免重复使用
- 时间戳偏移量控制，防止时间差导致的验证失败
### 敏感数据加解密
- AES 对称加密：用于敏感字段存储（如手机号、身份证）、接口传输加密
- RSA 非对称加密：用于密钥交换、签名验证
- 加密算法统一封装，提供 IEncryptionService 抽象接口
### 密码哈希
- 支持 PBKDF2、BCrypt、Argon2 等安全哈希算法，避免明文存储密码
- 哈希加盐处理，防止彩虹表破解
- 密码强度校验（长度、复杂度）
### 数据脱敏
- 自动脱敏：手机号（138****1234）、身份证（110****5678）、邮箱（a****@163.com）
- 脱敏规则可配置，支持自定义脱敏字段、脱敏格式
- 脱敏范围：日志输出、接口返回、导出文件
### 恶意请求防护
- IP 黑名单管理，拦截恶意 IP 请求
- 防暴力破解：登录、接口调用失败次数限制，触发限制后临时封禁
- 异常请求拦截（如恶意参数、超大请求体）
## 三、日志与诊断
### 结构化日志
- 基于 Serilog 实现结构化日志输出（JSON 格式）
- 日志级别控制（Debug/Info/Warn/Error/Fatal），按环境区分日志级别
- 日志模板标准化，包含必要字段（时间、日志级别、请求 ID、链路 ID、模块、消息）
### 请求日志
- 自动记录接口请求信息：请求路径、HTTP 方法、入参、出参、耗时、IP 地址、用户信息
- 忽略敏感参数（如密码、令牌），避免日志泄露
- 请求日志与链路追踪关联，便于排查问题
### 异常日志
- 捕获全局异常，记录完整堆栈信息、异常类型、异常消息
- 关联请求上下文，记录异常发生时的用户、租户、请求信息
- 异常分级记录，严重异常（Fatal/Error）触发告警
### 审计日志
- 记录关键操作：操作人、操作时间、操作对象、操作类型（新增 / 修改 / 删除）、操作前后值
- 审计日志不可篡改，支持追溯
- 按模块、操作人、时间范围查询审计记录
### 业务日志
- 记录核心业务事件：订单创建、支付完成、退款处理等
- 业务日志与业务流程关联，便于业务排查、数据统计
- 支持自定义业务日志模板
### 性能日志
- 监控慢查询（SQL 执行时间阈值）、慢接口（接口响应时间阈值）
- 记录外部服务调用耗时（如 HTTP 请求、MQ 消息发送 / 消费）
- 性能日志聚合分析，定位性能瓶颈
### 日志持久化
- 多渠道持久化：本地文件、Console 控制台、Elasticsearch、Loki
- 日志滚动策略（按大小、按时间），避免日志文件过大
- 日志留存策略，自动清理过期日志
## 四、监控与可观测性
### OpenTelemetry 全链路追踪
- 集成 OpenTelemetry，实现 Trace（链路追踪）、Span（跨度）、Baggage（上下文传递）
- 追踪微服务间调用链路，记录每个环节的耗时、状态
- 支持导出到 Jaeger、Zipkin、Prometheus 等追踪平台
### 应用指标 Metrics
- 核心指标监控：QPS、接口延迟、错误率、请求成功率
- 系统指标监控：GC 次数、GC 耗时、内存占用、线程数、CPU 使用率
- 业务指标监控：订单量、支付量、活跃用户数（可扩展）
- 指标导出到 Prometheus，支持 Grafana 可视化
### 健康检查
- 基础健康检查：应用存活状态、端口可用性
- 依赖健康检查：数据库、Redis、MQ、外部服务、配置中心
- 自定义健康检查：业务核心依赖（如缓存服务、存储服务）
- 提供健康检查端点（/health），支持 Kubernetes 存活 / 就绪探针
### 性能分析工具
- 集成 MiniProfiler，分析接口、SQL 执行性能
- 启用 .NET 诊断端口，支持实时性能分析、内存快照
- 支持性能分析数据导出，便于离线分析
### 基础告警能力
- 日志告警：基于日志级别、关键词触发告警（如 Error 日志、异常关键词）
- 指标告警：基于指标阈值触发告警（如错误率超过 1%、接口延迟超过 500ms）
- 告警渠道：邮件、短信、企业微信 / 钉钉机器人（可扩展）
## 五、缓存与分布式状态管理
### 内存缓存
- 基于 IMemoryCache 实现内存缓存，支持缓存过期策略（绝对过期、滑动过期）
- 层级缓存（内存缓存 + 分布式缓存），提升缓存命中率
- 缓存预热、缓存刷新机制，避免缓存穿透
### 分布式缓存
- 基于 Redis 实现 IDistributedCache，支持字符串、哈希、列表等缓存类型
- 缓存序列化（JSON / MessagePack），支持复杂对象缓存
- 缓存压缩，减少网络传输、节省内存
- 缓存防护：防穿透（空值缓存）、防击穿（互斥锁）、防雪崩（过期时间错开）
### 分布式锁
- 基于 RedLock.net 或 Lua 脚本实现分布式锁，支持公平锁、非公平锁
- 锁超时自动释放，避免死锁
- 应用场景：防并发竞争（如库存扣减）、防重复提交（如订单创建）
### 分布式限流
- 支持多种限流算法：固定窗口、滑动窗口、令牌桶、漏桶
- 限流粒度：全局限流、接口级限流、用户级限流、IP 级限流
- 限流策略可配置，支持动态调整限流阈值
- 限流触发后返回统一提示（如 429 Too Many Requests）
### 幂等性框架
- 基于幂等 Key（如请求 ID、订单号）实现接口幂等
- 幂等 Key 存储（Redis / 数据库），自动去重
- 支持多种幂等场景：接口重复调用、消息重复消费
### 分布式计数器与限频
- 基于 Redis 实现分布式计数器，支持原子增减
- 限频控制（如每分钟最多调用 100 次），与限流协同工作
- 计数器持久化，支持重启后恢复计数
## 六、高可用与弹性能力
### 熔断
- 基于 Polly 实现 CircuitBreaker 熔断模式
- 熔断触发条件：失败率超过阈值、连续失败次数达到阈值
- 熔断状态管理：闭合（正常）、打开（熔断）、半打开（试探）
- 熔断后兜底处理，避免级联故障
### 重试
- 基于 Polly 实现重试策略，支持指数退避、固定间隔、抖动重试
- 按异常类型重试（如网络异常、超时异常），避免无效重试
- 重试次数、间隔可配置，支持动态调整
### 舱壁模式（Bulkhead）
- 基于 Polly 实现舱壁模式，隔离不同业务的并发资源
- 限制每个业务的最大并发数，防止单个业务耗尽资源
- 避免级联故障，提升系统整体可用性
### 降级
- 核心接口降级策略：故障时返回兜底数据（如缓存数据、默认值）
- 非核心接口降级：故障时直接返回提示，停止业务处理
- 降级开关，支持手动 / 自动降级
### 超时控制
- 全局超时配置：所有外部调用、数据库操作设置默认超时
- 接口级超时配置，支持自定义超时时间
- 超时后自动中断请求，避免资源占用
## 七、数据访问基础设施（DDD 核心）
### 仓储模式 Repository
- 通用仓储接口 IRepository<T>，封装基础 CRUD 操作
- 聚合根仓储接口（如 IOrderRepository），封装聚合相关操作
- 仓储实现与 EF Core 解耦，支持替换数据访问框架
### 工作单元 UnitOfWork
- 统一事务管理，跨仓储事务提交、回滚
